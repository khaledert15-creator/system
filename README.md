# نظام إدارة مكتبة دوت كوم

نسخة Windows محلية تشغيلية لإدارة الكتب والمخزون والمبيعات والمشتريات والعملاء والموردين والخزينة والشحن والتقارير والموظفين.

## التشغيل

### التشغيل الموحد باستخدام Docker

لا توجد بيئات Test وProduction منفصلة. بعد Clone يمكن تشغيل النظام مباشرة دون إنشاء أو تعديل أي ملف:

```bash
docker compose up -d
```

سيتم بناء وتشغيل النظام وقاعدة PostgreSQL وخدمة Omnichannel، مع حفظ البيانات داخل Docker Volumes.

ملف `.env` اختياري. عند تجهيز Domain أو ربط WhatsApp/Messenger، أنشئه من المثال لتخصيص الروابط والأسرار:

```bash
cp .env.example .env
```

لمراجعة الحالة:

```bash
docker compose ps
docker compose logs --tail=100 omnichannel
```

إيقاف الخدمات دون حذف البيانات:

```bash
docker compose down
```

لا تستخدم `docker compose down -v` لأنه يحذف Volumes وقواعد البيانات.

### التشغيل المباشر على Windows

1. فك ضغط الحزمة في مجلد عادي على Windows.
2. اضغط مرتين على `START-HERE.cmd`.
3. يبدأ خادم قاعدة البيانات في الخلفية، ثم يفتح النظام تلقائيًا على:

   `http://127.0.0.1:8765/`

لا تفتح `index.html` مباشرة، لأن الحفظ الدائم على القرص يحتاج تشغيل الخادم بواسطة `START-HERE.cmd`.

## الدخول الأول

- اسم المستخدم: `owner`
- كلمة المرور الأولية: `DotCom@2026`

توجد أيضًا حسابات: `manager` و`accountant` و`cashier` و`warehouse` و`shipping` بنفس كلمة المرور الأولية. غيّرها قبل التشغيل الفعلي.

## قاعدة البيانات

- تُحفظ البيانات في `data/database.json`.
- يتم إنشاء نسخة احتياطية قبل كل عملية حفظ.
- توجد النسخ الاحتياطية في `data/backups`.
- يحتفظ النظام بأحدث 30 نسخة لتجنب امتلاء القرص.
- يمكن إنشاء نسخة احتياطية يدويًا من شاشة الإعدادات.
- يوجد حفظ احتياطي داخل المتصفح إذا تعذر الاتصال بالخادم، مع ظهور تنبيه واضح.

## الوظائف المكتملة

- لوحة متابعة وتنبيهات المخزون والشحن.
- بطاقات لوحة المتابعة تفاعلية؛ تعرض تفاصيل المبيعات والمخزون والمديونيات والأصناف الحرجة.
- إجراءات مباشرة من لوحة المتابعة مثل فتح الفاتورة، التحصيل، كشف الحساب، التسوية وتعديل الصنف.
- مركز التنبيهات تفاعلي، ويعرض تفاصيل كل تنبيه مع أزرار تعديل واتخاذ إجراء.
- يمكن إنشاء مستند شراء مباشرة من تنبيه انخفاض المخزون، أو تعديل الشحنة من تنبيه التتبع.
- الضغط على تفاصيل أو تعديل من التنبيهات ينقل مباشرة للسجل المطلوب ويميزه بأنيميشن ذهبي.
- تنبيه الشحنة يتيح فتح الفاتورة المرتبطة مباشرة مع إبرازها قبل عرض تفاصيلها.
- قائمة «تنبيهات تحتاج قرارًا» داخل لوحة المتابعة نفسها قابلة للنقر وتعرض أزرار التفاصيل والتعديل والإجراء المناسب.
- زر عدد التنبيهات يفتح مركز المتابعة الكامل، وتدعم عناصر القائمة التشغيل بلوحة المفاتيح.
- بطاقة الشحنات الجارية تتزامن تلقائيًا مع قاعدة البيانات كل دقيقة دون إعادة تحميل الصفحة.
- يظهر وقت آخر تحديث، مع زر «تحديث الآن» وفتح تفاصيل الشحنة مباشرة من لوحة المتابعة.
- نموذج إضافة وتعديل الكتاب يحتوي على سعر المورد وخصم الشراء، ويحسب تكلفة الشراء الصافية تلقائيًا.
- الجرد الجزئي والكلي يعملان فعليًا مع اختيار الرف أو التصنيف أو الأصناف يدويًا، وحساب الفروقات واعتمادها دفعة واحدة.
- تسجيل دخول بجلسة خادم وأدوار مالك ومدير ومحاسب وكاشير ومخزن وشحن.
- منع الوصول إلى الصفحات والإجراءات حسب الدور، مع حماية API.
- سجل حركة مخزون لكل كتاب مرتبط بالمستند والمستخدم والرصيد قبل وبعد.
- قسم طلبات أونلاين مستقل مع التحويل إلى فاتورة وشحنة وتنبيهات التأخير.
- بحث مبيعات بالباركود أو الاسم ودمج الكميات المتكررة.
- قوالب طباعة A4 وحرارية وتقارير قابلة للتصدير CSV.
- استعادة النسخ الاحتياطية ومنع تعارض الحفظ بين أكثر من نافذة.
- إضافة وعرض وتعديل وحذف الكتب غير المرتبطة بحركات.
- أكثر من باركود، مورد، رف، تكلفة، سعر، رصيد وحد إعادة طلب.
- الجرد وتسوية التالف والمفقود والنسخ المجانية.
- إنشاء فواتير بيع وحساب الإجمالي والخصم والمدفوع والمتبقي.
- البحث في الفواتير القديمة برقم الفاتورة أو كود تتبع الشحنة أو رقم الموبايل أو اسم العميل.
- عرض كود التتبع وحالة الشحنة داخل نتائج بحث الفواتير.
- منع البيع بأقل من التكلفة.
- تحديث المخزون تلقائيًا بعد البيع وإرجاعه عند إلغاء الفاتورة.
- تسجيل التحصيلات الجزئية وتحديث مديونية العميل والخزينة.
- إنشاء مشتريات أو أمانة وحساب الشحن والمدفوع والمتبقي.
- تحديث المخزون بعد الاستلام، وعدم تحديثه قبل اعتماد الفحص.
- إلغاء مستندات البيع والشراء مع عكس أثر المخزون والحسابات.
- إضافة وعرض وتعديل وحذف العملاء والموردين وفق قواعد سلامة البيانات.
- إيصال استلام وإيصال دفع خاص بكل عميل أو مورد.
- ربط الإيصال بالخزنة وكشف حساب الطرف مع تسوية المديونية أو تسجيل دفعة مقدمة.
- عرض الإيصال وطباعته وإلغاؤه مع عكس أثره المالي تلقائيًا.
- سندات قبض وصرف وتصنيف المصروفات وتعديلها وحذفها.
- شركات شحن، أكواد تتبع، حالات الشحن والإرجاع.
- تقارير المبيعات والربحية والمخزون والركود والمديونيات والخصومات والشحن.
- الموظفون والرواتب والحضور ونموذج الصلاحيات.
- سجل عمليات ونسخ احتياطي ورسائل نجاح وخطأ عربية.

## قواعد الحذف

لحماية الحسابات، لا يسمح النظام بحذف كتاب أو عميل أو مورد مرتبط بفواتير. يجب إلغاء الفاتورة المعتمدة أولًا، ثم يمكن حذف المستند الملغى من العرض، بينما يبقى الحدث محفوظًا في سجل العمليات.

## حدود النسخة الحالية

هذه قاعدة بيانات ملفية محلية مناسبة للتجربة والتشغيل على جهاز واحد. التشغيل الفعلي عبر الإنترنت لعدة محاسبين في الوقت نفسه يحتاج المرحلة السحابية:

- قاعدة PostgreSQL مركزية.
- تسجيل دخول وكلمات مرور مشفرة.
- صلاحيات تُفرض من الخادم.
- سجل تدقيق غير قابل للتعديل.
- استضافة HTTPS ونسخ احتياطي خارجي.
- ربط منصة المتجر وشركات الشحن وWhatsApp Business.
- مراجعة محاسب لدليل الحسابات والقيود القانونية.
- ربط الفاتورة أو الإيصال الإلكتروني عند الانضمام للمنظومة المصرية.

## إيقاف الخادم

الخادم يعمل في الخلفية على المنفذ `8765`. يتوقف عند إغلاق عملية PowerShell الخاصة به أو إعادة تشغيل Windows، ويعاد تشغيله تلقائيًا في المرة التالية عند الضغط على `START-HERE.cmd`. البيانات تظل محفوظة على القرص.
