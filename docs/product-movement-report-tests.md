# اختبارات تقرير حركة صنف

تاريخ التنفيذ: 2026-07-19  
الفرع: `feature/product-movement-report`

## اختبارات آلية ومنطقية

- فحص JavaScript بواسطة `node --check`: ناجح.
- فحص المسافات وأخطاء patch بواسطة `git diff --check`: ناجح.
- اختبار معادلة الرصيد لكل صنف: `أول المدة + الداخل - الخارج = آخر المدة`.
- اختبار تصنيف كل حركة إلى شراء أو بيع أو مرتجع أو تسوية أو جرد أو إلغاء.
- اختبار أن حجم الصفحة ثابت عند 25 حركة.
- اختبار صلاحية التكلفة: المستخدم بدور كاشير لا يحصل على `view-item-cost-profit`، وبالتالي لا تظهر أعمدة التكلفة والربح.

يمكن تشغيل الاختبارات المنطقية من Console الصفحة عبر:

```js
runProductMovementReportTests()
```

## اختبار واجهة فعلي

| الحالة | النتيجة |
|---|---|
| فتح التقرير من قسم التقارير | ناجح |
| اختيار الصنف بالكود | ناجح |
| عرض ملخص الفترة | ناجح |
| معادلة الصنف B001 | ناجح: 22 - 2 = 20 |
| فلتر نوع الحركة: مبيعات | ناجح |
| فتح المستند من صف الحركة | ناجح: فُتحت INV-1049 |
| ظهور فلاتر المورد والعميل والموظف والحالة | ناجح |
| اتجاه الصفحة RTL | ناجح |
| مقاس 1440×900 | ناجح، دون overflow على مستوى الصفحة |
| الجدول الواسع | ناجح، التمرير الأفقي محصور داخل الجدول |
| زر الطباعة | ظاهر ومربوط بإخراج A4 |
| زر CSV | ظاهر ومربوط بتصدير UTF-8 BOM |

## اختبارات القبول بعد الدمج

1. إعادة تجربة الطباعة على Chrome مع السماح بالنوافذ المنبثقة.
2. فتح ملف CSV في Excel والتحقق من العربية والأرقام.
3. تجربة التقرير بحساب مالك ثم بحساب كاشير للتحقق بصريًا من إخفاء التكلفة.
4. اختبار صنف يتجاوز 25 حركة للتأكد من السابق/التالي.
5. اختبار صنف بلا حركات، ومستند قديم محذوف، وحركة تسوية بلا فاتورة.
